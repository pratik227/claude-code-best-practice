# Changelog

This section tracks the drift audit history for each best practice report. Every changelog records what changed between Claude Code versions, what actions were taken, and their resolution status.

## Reports

| Report | Description |
|--------|-------------|
| [Commands](./commands) | Built-in slash commands -- new commands, removed commands, argument changes, and description syncs |
| [Skills](./skills) | Skill frontmatter fields and bundled skills -- field additions and accuracy checks |
| [Subagents](./subagents) | Subagent frontmatter fields, built-in agents, and invocation methods |
| [Settings](./settings) | Settings keys, environment variables, permissions, sandbox, and hierarchy changes |
| [Concepts](./concepts) | README CONCEPTS table -- missing concepts, broken URLs, badge accuracy, and description drift |
| [Development Workflows](./development-workflows) | Community workflow repos -- star counts, feature counts, and sort order updates |

## How to Read These Changelogs

Each changelog entry is tagged with a date, Claude Code version, and a table of action items.

::: info Status Legend
- **COMPLETE** -- Action was taken and resolved successfully
- **INVALID** -- Finding was incorrect, not applicable, or intentional
- **ON HOLD** -- Action deferred, waiting on external dependency or user decision
:::

::: tip Priority Levels
- **HIGH** -- New fields, missing items, broken links, or behavioral changes requiring immediate action
- **MED** -- Description updates, annotation fixes, version bumps
- **LOW** -- Cosmetic changes, verification confirmations, suspect key tracking
:::
